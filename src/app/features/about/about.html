Smart Assist – User Manual

Module 2: Introduction to Angular

1. About Smart Assist

Smart Assist is an Angular-based learning application designed to help users understand the fundamentals of Angular and enterprise-ready application development.

This project is part of Module 2 – Introduction to Angular and focuses on building a functional Angular application with proper structure, routing, and UI.

2. Module 2 – Learning Objectives

  • Set up Angular development environment
  • Understand Angular CLI and project scaffolding
  • Learn Angular application bootstrapping
  • Create and organize components
  • Implement basic routing
  • Use Angular Material for UI design
  • Follow enterprise-ready Angular architecture

3. Project Structure (Enterprise-Ready)

The Smart Assist application follows a feature-based, scalable Angular architecture. 

src/
 └── app/
     ├── core/        (Services, guards)
     ├── shared/      (Reusable components)
     ├── auth/        (Login module)
     ├── dashboard/   (User dashboard)
     ├── manual/      (User manual page)
     ├── app-routing.module.ts
     └── app.module.ts

Key principles used:

  • Feature-based modules
  • Single Responsibility Principle
  • Clean separation of concerns
  • Enterprise-ready structure

4. Home Page Components

The Home Page is built using three main components:

  • Header Component – Displays the application title Smart Assist
  • Login Component – Allows users to log in using hardcoded credentials
  • Footer Component – Displays developer info and a User Manual link

5. Login Functionality

The login feature is implemented using hardcoded credentials for demonstration purposes.

Sample Credentials:

  • Username: user
  • Password: user123

On successful login, the user is redirected to the User Dashboard Main Page.

6. User Dashboard

The User Dashboard is displayed after successful login and serves as the main landing page for the user.

  • Confirms successful authentication
  • Demonstrates routing functionality
  • Acts as a base for future feature expansion

7. Angular Concepts Covered

Data Binding

  • Interpolation
  • Property Binding
  • Event Binding
  • Two-Way Binding

Component Communication

  • @Input()
  • @Output() with EventEmitter

8. Routing & Navigation

Angular Router is used to navigate between pages in the application.

  • /login – Login page
  • /dashboard – User dashboard
  • /manual – User manual page

9. UI & Styling

Angular Material components are used to build a clean and responsive UI.

  • Toolbar
  • Buttons
  • Cards
  • Form Fields

10. Evaluation Criteria

AC#1 – Functional Application

  • Login page is available and functional
  • Hardcoded login works correctly
  • Routing is implemented
  • User dashboard loads after login
  • Responsive UI

AC#2 – Code Quality & Repository

  • Public GitHub repository
  • Proper Angular project structure
  • Minimum 2–3 meaningful commits
  • Project runs without errors

11. How to Run the Application

npm install
ng serve

Open the browser and navigate to: http://localhost:4200
